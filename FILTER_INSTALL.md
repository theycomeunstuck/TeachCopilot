# FILTER_INSTALL.md — Установка Filter Function в Open WebUI

## Шаг 1: Откройте Open WebUI

Зайдите в Open WebUI в браузере (обычно `http://localhost:8080`).

## Шаг 2: Перейдите в раздел Functions

1. Нажмите на **Workspace** в боковом меню
2. Выберите вкладку **Functions**
3. Нажмите кнопку **+ New Function** (или значок «+»)

## Шаг 3: Настройте функцию

1. **Name**: `TeachCopilot Filter`
2. **Description**: `Personalized AI tutor — injects child profile + RAG into system prompt`
3. **Type**: выберите **Filter**

## Шаг 4: Вставьте код

Скопируйте и вставьте содержимое файла `pipeline/filter_function.py`:
- Все строки `import` в начале
- Весь класс `Filter` с методом `inlet`
- **НЕ** копируйте docstring модуля (тройные кавычки в начале файла)

Начните копирование с:
```python
import os, re, logging
from pipeline.db import get_child_full_profile
...
```

И до конца файла.

## Шаг 5: Включите фильтр для модели

1. Перейдите в **Workspace** → **Models**
2. Выберите модель, которую используете (например, Claude или LLM через LM Studio)
3. В настройках модели найдите раздел **Filters**
4. Включите `TeachCopilot Filter`
5. Сохраните

## Шаг 6: Проверьте работу

Отправьте сообщение в чат:

```
[child_id:00000000-0000-0000-0000-000000000001] объясни отрезок
```

### Что должно произойти (успех):

- Фильтр извлечёт `child_id` из префикса `[child_id:...]`
- Загрузит профиль ребёнка (Миша, 9-11 лет, 3-4 класс)
- Найдёт релевантный материал по RAG (геометрия: отрезки)
- Соберёт системный промпт с профилем и материалом
- LLM ответит персонализированно: простым языком, с примерами, учитывая что Миша путает луч и отрезок

### Что означает ошибка:

- Если ответ **не персонализирован** (без упоминания имени, без адаптации) — фильтр не подключён к модели. Проверьте шаг 5.
- Если появляется **ошибка импорта** — убедитесь, что `pipeline/` доступен в PYTHONPATH сервера Open WebUI.
- Если RAG **не находит материал** — проверьте, что `python scripts/ingest.py --dir data` был выполнен на сервере.

## Без префикса child_id

Если отправить сообщение без `[child_id:...]`, фильтр использует `DEFAULT_CHILD_ID` из `.env`.
Если `DEFAULT_CHILD_ID` не задан — промпт будет базовым (без персонализации).
